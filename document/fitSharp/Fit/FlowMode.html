Flow Mode
In Flow Mode, a fixture processes all the tables in the story test, without repeating the fixture name in the first row of each table.

Each row is interpreted as one of the following.
  • One of the standard Flow Fixture Keywords.
  • A custom flow fixture keyword defined in the current fixture.
  • A method defined in the current fixture.
  • A method defined in the current fixture's System Under Test.
  • The name of a new fixture that will process the rest of the current table.

If the method invoked for a row returns certain types, the return value will be interpreted by a new fixture that will process the rest of the table.

Any enumerable return type is processed by an Array Fixture.

when test is
sample domain
get peopleBob-Martin,Mike-Stockdale
first namelast name
BobMartin
MikeStockdale
then result is
sample domain
get peopleBob-Martin,Mike-Stockdale
first namelast name
BobMartin
MikeStockdale

To use a different fixture for an enumerable return type, use the Use Fixture.

when test is
sample domain
usesetwithget peopleMike-Stockdale,Bob-Martin

MikeStockdale
then result is
sample domain
usesetwithget peopleMike-Stockdale,Bob-Martin

BobMartin
MikeStockdale

when test is
sample domain
usesubsetwithget peopleMike-Stockdale,Bob-Martin

BobMartin
then result is
sample domain

first namelast name
